깃 명렁어

$ add ->
git add {파일이름 or 폴더명} // 선택된 파일을 스테이지에 올리는 작업이다.

$ commit ->
git commit -m {메시지} // 레포지터리에 파일 커밋한다. 여기서 {괄호 안 메시지는}
레포지토리에 커밋 할때 표시할 내용을 정하는 것이다.
git commit -a // 신규 파일을 제외하고 변경 사항을 스테이징하고 커밋하는 것이다.
git commit —amend // 이전에 커밋한 것을 변경하는 명령어

$ diff ->
diff 명렁어는 commit 이나 branch 사이 다른점 또는 파일이나 Repository 와 Working Directory 사이 다른 점을 보여주는 명령어다.
git diff {옵션 명} {파일 이름} // 특정한 파일에 대한 변경 사항을 확인하는 명령어이다.

$ pull -> pull 명령어는 저장소에 로컬 저장소로 데이터를 가져오는 명령어다.
git pull // 클라우드 저장소에 있는 데이터를 로컬 저장소로 가져오는 명령어다.

$fetch ->
fetch 는 로컬 저장소와 클라우드 저장소에 최신 데이터가 있는지 확인하는 명령어이다.

$push ->
git push // 로컬 저장소에 있는 파일을 git 클라우드 저장소에 보내는 명령어다.

$reset ->
reset 명령어는 예전 커밋으로 돌아가고 싶을때, 또는 오류로 이전 커밋으로 롤백 할 때 사용하는 명령어다
git reset —hard {커밋명} // 커밋이 생성될 때로 시간을 되돌린다.

$ restore -> restroe 명령어는 과거 커밋 내용을 되돌리는 명령어다.
git restore {파일명} // 가장 최근 커밋된 상태로 파일 내용을 수정한다.

$status -> status 명령어는 git add, git commit 명령어들과 고나련하여 무슨 일이 일어났는지 보여주는 명령어다. 스테이징된 파일, 스테이징 되지 않은 파일, 추적되지 않을 파일을 나열한다.
$log -> log 명령어는 커밋 된 기록을 조회하는 명령어다.
git log // 커밋된 기록을 조회하는 명령어
git log —stat // 각 커밋의 통계를 볼 수 있다. 파일, 줄, 수정, 추가, 삭제 되었는지 보여주는 명렁어다.

용어 정리

$CLI ->
CLI는 Command Line Intercafe의 약자다. 흔이 아는 DOS, PowerSehll, BashSehll, zshShell 등이 있다.
CLI환경의 장점으론 컴퓨터 자원을 적게 소모해 안정성이 높고성능이 빠르다. 네트워크 환경 사용시 데이터 양이 적음,
단점은 CLI환경에 익숙해지기까지 시간이 걸린다. 기본적인 명령어를 외워야한다.

$GUI ->
GUI는 Graphical User Interface의 약자로 흔이 아는 MAC os, 윈도우스 처럼 사용자가 글과 그림으로 이루어진 환경에서 사용하는 것을 말함.
장점은 CLI에 비해 눈에 확 들어와 현제 무엇을 하고 있는지 알 기 쉽고, 조작 방법이 직관적이다. CLI환경과 다르게 마우스 커서를 이용해 파일을 열고 닫을 수 있다.
단점으로는 컴퓨터 자원을 많이 사용한다. 특히 원격 접속 시 그래픽 파일을 전송하기에 데이터 소모가 크다.

$WOL ->
Wake on LAN 의 약자로 PC를 원격으로 켜는 기능이다.
WOL을 사용하기 위해서는 PC에 유선 LAN이 꼭 연걸되어아한다.
LAN 선이 컴퓨터를 종료해도 항상 외부와 연결되어있기에 해킹 공격에 취약하다.

$AC부팅 -> 컴퓨터에 AC전원이 연결되면 자동으로 부팅되게 하는 기능이다.
바이오스 UEFI 에서 설정이 가능함.

$쓰레드 ->
쓰레드란 프로세스 내에 제로 작업하는 주체를 말한다.
-여기서 프로세스란 실행 중인 프로그램이라고 할 수있다.
프로세스 내에서 실행되는 흐름의 단위, CPU 스케줄링의 기본 단위 라고 할 수 있다.

$문맥 교환 ->
문맥 교환이란 프로세스가 CPU를 사용 중인 상태에서 다른 프로세스가 CPU를 사용하도록 하기 위한 , 일종의 JUMP를 할때 기존 상태(문맥)을 보관하고 다른 프로그램 상태를 적제하는 작업을 말 함.

$REST-API ->
REST-API 란 REST원리를 따르는 API를 의미한다.
여기서 REST란 자원을 이름으로 구분하여 해상 다원의 상태를 주고 받는 모든 과정을 말한다.
REST의 구성 요소는 자원, 행위, 내용 이 있다.

종합해보면 REST는 HTTP 메소드를 통해 자원을 처리하도록 설계된 아키텍쳐이다.
그럼 REST-API란 REST를 기반으로 API서비스를 구현한 것이다. 예로 웹에서 특정 기술? 을 사용하고 싶은때 REST 규격을 사용해 API를 사용하는 것을 말한다.

$에자일 기법 ->
반복 작업을 통해 실제 작동 가능한 소프트웨어를 개발하여 지속적으로 제공하기 위한 소프트웨어 개발 방식
주로 대규모 프로젝트에 사용하고 크고 복잡한 프로젝트에 사용됨

$폭포수 기법 ->
개발 단계를 폭포수 처럼 위에서 아래로 물 떨어지듯 차례대로 진행하는 과정이다. 다른 단계와 병행해서 진행하거나 거꾸로 진행되는 경우는 없다
주로 소규모 프로젝트, 개인 개발자들이 사용함

$HTML -CSS -JS ->
웹 프론트 언어들로서 가장 기본이 되는 언어이다. HTML은 웹 문서의 뼈대를 구측하는 개념이다.
CSS는 기존 HTML파일에 다양한 모양을 추가 변경하여 웹사이트의 이미지를 구측 하는 것이다.
JS는 웹에 움직임을 넣는 언어어이다.

$nginx->
Nginx는 트래픽이 많은 서버를 도와주는 이벤트 기반 구조의 경량 웹 서버 프프로그램이다.
클라이언트로 부터 요청을 받앗을때 정적 파일을 응답해주는 HTTP 웹서버로 활용 되기도 하고 리커버리 프록시 서버로 활용하여 AWS의 부하를 줄일 수 있는 로드밸런서 역할을 하기도 함

$ IP -> Internet Protocol을 약자로 네트워크에서 수진 송신하는 규격을말함
IPv4 // 널리 사용 되는 주소로 우리가 흔이 아는 IP주소다.
IPv6 // 128비트 값을 가지는 주소로 IPv4 주소 고갈을 앞두고 차기 주고 체계로 고안됨
공인 IP // 국가별로 주어진 IP 주소이다. 어떤 IP를 쓰고 싶을 때 IP주소를 관리하는 기관에 사용 요청을 받아야 사용할 수 있음
사설 IP // IP주소를 할당 받기 위해 비용을 지불 해야 하는데 다량으로 IP확보가 어려울 때 쓰는것. 흔이 하는 공유기를 생각하면 된다.

$TCP 와 UDP ->
TCP는 데이터 전송 방식의 종류 로 데이터를 전송할 때 보안에 좋고 안정적이다. 대신 속도가 느리다
UDP 는 데이터 속도가 빠른 대신 페킷 손실이 일어난다.

$DHCP ->
DHCP란 호스트IP의 주소와 각종 프로토콜의 설정을 클라이언트에게 자동으로 제공해주는 프로토콜을 말한다.

$DNS ->
DNS란 IP주소 를 문자로 변환해 사용하는 것을 말 함.
대표적으로 216.58.200.14는 구글 닷검의 아이피 주소로 DNS로 표연하면 www.google.com 으로 표현 가능하다.

$리눅스와 유닉스 ->
리눅스와 유닉스의 차이는 유닉스는 독점 운영 체제이고 리눅스는 오픈 소스 운영체제다. 유닉스는 누구던 수정 사용할 수 있지만 유닉스는 구입해야함
유닉스의 경우 대형 컴퓨터에 사용되는 반면 리눅스는 작거나, 개인 컴퓨터, 서버급 PC에서 사용된다.
유닉스에서 사용 되는 코드들은 여전히 리눅스에서 사용되는 중이다. 리눅스는 유닉스 기반으로 하기 때문

$UI/UX ->
UI 유저 인터페이스 사용자가 제품을 사용할 때 마주하게 되는 것
UI는 시각적 디자인, 즉 컬러 폰트 레이아웃 등과 같은 것을 표연하는 방법이다.
UX는 사용자게 제품을 사용하면서 느끼는 종합적인 만족도를 말한다.
